## 1 章
- functors（ファンクタ）
- monoids（モノイド）
- monads（モナド）
- FP で重要なのは不変性
- FP では再帰関数が用いられる
- referential transparency（参照透過性）
    - ある式が参照透過であるとは、その式をその値で置き換えてもプログラムの意味が変わらないことを意味する??
    - 日本語が難しすぎるやろ
- 「純粋」とは
    - 副作用がないこと
    - 入力に対して同じ出力を返すこと
    - 参照透過性と純粋は類義語として混同される
- lazy evaluation（遅延評価）
    - 遅延評価は、式が評価されるまでその値が計算されないことを意味する
    - 例えば条件分岐とかで片方しか評価せずにすむ
- FP の簡単な説明と FP はいいですよ！という話

## 3 章 Category Theory and Patterns

- FP におけるパターンは OOP のパターンとはことなって、圏論由来のものがある
- Option パターン
    - ある値が存在するかどうかを表す
    - ある値が存在しない場合には、None という値を返す
    - ある値が存在する場合には、Some(hoge) という値を返す
        - これ完全に Rust の Option と同じやんけ。ここで Rust とつながるとは
        - ここ進研ゼミでやったところだ！状態
- Functors
    - ある圏からある圏を結ぶ関数
    - F(f o g) = F(f) o F(g)を満たす
    - プログラミングにおいては、map を実装したインタフェース（トレイト）
- Monoid
    - ある集合とその集合上の演算子がモノイドであるとは、その演算子が結合法則を満たし、単位元を持つことを意味する
    - 例えば、加算演算子と 0 はモノイドである
    - プログラミングにおいては、モノイドは、ある型とその型上の演算子がモノイドであることを意味する
