3 版も出版されている時点でとても良い書籍であることはいうまでもないですが、とても良い本でした。

1 章 ブラウザは何をしているのか？

Get メソッドや POST メソッドで何が起きているのか、そして Cookie がどんな風にやり取りされるのか簡単に説明されている。

- HTTP はクライアントからサーバーに対して、情報をリクエストしてその情報をもらうためのプロトコル

2 章 HTTP/1.0 の世界：基本となる 4 つの要素

- 4 要素は以下
    - メソッド＆パス
    - フィールド（ヘッダ）
    - コンテンツ
    - ステータス

- 書籍を通して、curl を使ってブラウザとのやり取りをする都合上 curl の解説も結構豊富にあるので、curl 初心者が読むのには結構いいかも
- url フラグメントテキストディレクティブとか初耳　https://wicg.github.io/scroll-to-text-fragment
- 3 版では localhost で公開してしまった例などが載せられていて面白い
- ホモグラフ攻撃

3 章

- cookie
    - Expires は Wed, 09 Jun 2021 10:18:14 GMT のような形式で指定
    - Max-Age は秒数で指定
    - Domain
    - Path
- Proxy

- cache
    - Expires は HTTP/1.0 で導入された。日時を格納してこの期限内であればキャッシュを使う
    - Etag はリソースのハッシュ値を格納しておいて、リソースが変更されていないかを確認する
    - Cache-Control は Expires よりも優先される

8 章
- HTTP/2 は TLS が必須
    - HTTP/2 は SPDY（グーグルが作ったプロトコル）をベースにしている
    - HTTP/2 はテキストベースからバイナリベースになったよ
    - HTTP/3 は QUIC をベースにしている
    - websocket はステートフル

14 章
- DNS ラウンドロビンとは一つのドメインに対して複数の IP アドレスを返すこと。これにより負荷分散ができる
- ドメイン名と IP アドレスの対応は A レコード。IPｖ６ は AAAA レコード
- ドメイン名から別のドメイン名を参照することを CNAME レコード（canonical name）
- プロキシは内部から外部への通信を中継するもの。リバースプロキシは外部から内部への通信を中継するもの
- リバースプロキシにも種類がある。コンテンツをキャッシュしたら CDN、負荷分散したらロードバランサ、セキュリティ対策したら WAF
- ingress は nginx ベースのリバースプロキシ
